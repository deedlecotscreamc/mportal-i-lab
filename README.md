# mportal-i-lab

## 项目简介

mportal-i-lab 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或具体网站，仅作为 HTML 页面资源的集合存储与版本管理用途。

## 目录说明

```
mportal-i-lab/
├── pages/          # 存放独立 HTML 页面文件
├── assets/         # 静态资源文件（CSS、JS、图片等）
├── docs/           # 文档与说明文件
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹或独立文件对应一个归档页面。
- `assets/` 用于存放页面所需的共用资源。
- `docs/` 可包含页面使用说明或归档记录。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，彼此之间无依赖关系。每个页面可单独打开浏览，部分页面可能引用 `assets/` 下的资源文件。

页面归档遵循以下原则：
- 每个页面保持自包含或引用仓库内资源。
- 页面内容不包含外部跟踪、广告或第三方服务。
- 历史版本可通过 Git 提交记录追溯。

## 维护说明

- 新增页面：在 `pages/` 下创建新文件夹或文件，并更新本 README 的目录说明（如有必要）。
- 更新页面：直接修改对应 HTML 文件，并提交变更。
- 删除页面：移除对应文件或文件夹，并更新相关引用。
- 资源管理：共用资源请放置于 `assets/` 对应子目录中。
